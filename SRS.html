סקירה כללית

General Overview

1.1 מטרת המערכת

מערכת ניהול אירוע רפואי המיועדת למנהלי אירוע רפואי (Case Managers) לניהול מסע המטופל מרגע האבחון ועד ההחלמה, כולל ניהול ביטוחים, תביעות, מסמכים, ואסטרטגיה פיננסית-רפואית.

1.2 קהל יעד

תפקידתיאורהרשאות
Admin — מנהל ראשיהבעלים של המערכתגישה מלאה + ניהול משתמשים
Manager — מנהל אירוע רפואיאיש מקצוע המלווה מטופליםניהול מטופלים ותהליכים
Patient — מטופלהלקוח הסופיצפייה במסע האישי בלבד

1.3 טכנולוגיות

שכבהטכנולוגיההערות
BackendFastAPI (Python 3.13), SQLAlchemyRESTful API
DatabaseSQLiteRailway Volume — פרסיסטנטי
FrontendReact 18 + Vite + Tailwind CSSRTL מלא
AuthJWT + 2FA (TOTP / Email)bcrypt, python-jose
DeployRailway (Production)GitHub Actions CI/CD
Source ControlGit / GitHubAuto-deploy on push to main

3.1 מנוע זרימת עבודה — Flow Engine ⭐

⭐
עדיפות גבוהה ביותר — המודול הבא בתכנון. ראה מסמך FLOW_ENGINE.md לפירוט מלא.

מושגי יסוד

מושגתיאור
Workflow Templateתבנית זרימה ניתנת לשימוש חוזר (למשל: "הגשת תביעה לקופ"ח")
Workflow Instanceהרצה ספציפית של תבנית עבור מטופל מסוים
Step / Stateשלב בזרימה (למשל: "איסוף מסמכים", "הגשה", "המתנה")
Transitionמעבר בין שלבים — עם תנאים / טריגרים
Actionפעולה אוטומטית או ידנית הנדרשת בשלב
Assignmentשיוך שלב למשתמש / תפקיד

תבניות מוגדרות מראש

WF-1
הגשת תביעת ביטוח — איסוף מסמכים ← הגשה ← מעקב ← תוצאה
WF-2
קבלת חוות דעת שנייה — חיפוש רופא ← קביעה ← קבלת חוות דעת ← שילוב בתיק
WF-3
ערר על דחייה — ניסוח ← הגשה ← שמיעה ← תוצאה
WF-4
תהליך אשפוז — רישום מוקדם ← כניסה ← שחרור ← מעקב
WF-5
הגשה לביטוח לאומי — בדיקת זכאות ← איסוף טפסים ← הגשה ← ועדה רפואית

דרישות לא פונקציונליות

Non-Functional Requirements
NFR-01
אבטחה: JWT עם תפוגה, 2FA חובה לכל משתמש, HTTPS ב-Production
NFR-02
ביצועים: תגובת API פחות מ-500ms לשאילתות רגילות
NFR-03
זמינות: 99.5% uptime דרך Railway
NFR-04
נגישות: ממשק RTL מלא בעברית, תמיכה ב-Chrome / Firefox / Edge
NFR-05
אחסון: Volume מתמיד ב-Railway לDB ולמסמכים
NFR-06
גיבוי: גיבוי DB אוטומטי — לא מיושם עדיין
NFR-07
הרחבות: ארכיטקטורה מודולרית — הוספת routes/modules ללא שינוי ליבה
NFR-08
CI/CD: כל push ל-main מפרס אוטומטית ל-Railway דרך GitHub Actions

מונחים והגדרות

Glossary
מנהל אירוע רפואי
איש מקצוע המלווה מטופל לאורך תהליך רפואי מורכב
מסע מטופל
ציר הזמן של האירוע הרפואי מאבחון ועד החלמה
תביעה
בקשה לקבלת פיצוי/כיסוי מחברת ביטוח או קופת חולים
זכאות
זכות כספית או שירותית שמגיע למטופל
צומת (Node)
אירוע בודד בציר הזמן של המטופל
כיסוי
תנאי הביטוח לקטגוריה מסוימת
Workflow
תהליך מובנה עם שלבים, אחראים, ודדליינים
Flow Engine
מנוע State Machine להגדרה והפעלה של workflows
2FA
אימות דו-שלבי — TOTP (Google Authenticator) או אימייל
JWT
JSON Web Token — מנגנון אימות stateless
