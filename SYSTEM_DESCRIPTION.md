## תקציר מנהלים

**Orly Medical** היא פלטפורמת ניהול אירוע רפואי (Medical Case Management) המיועדת לארגונים המספקים ליווי רפואי מקצועי למטופלים בתהליכים מורכבים — אונקולוגיה, ניתוחים, שיקום, ומחלות כרוניות.

### הבעיה שאנו פותרים

מטופל עם אירוע רפואי קשה מתמודד בו-זמנית עם עשרות גורמים: רופאים מתמחויות שונות, ביטוחים מרובים, תביעות, מסמכים, תורים ומסמכים. מנהלי האירוע הרפואי מנהלים תהליכים אלה ידנית — ספרדשיטים, מיילים, ותיקיות — ללא מערכת מרכזית.

### הפתרון

מערכת אחת המאגדת את כל מרכיבי ניהול האירוע הרפואי: מסע המטופל, ביטוחים ותביעות, מסמכים, מאגר רופאים, וזרימות עבודה מובנות — עם אוטומציה, ממשק RTL בעברית, ותמיכה מלאה במובייל ו-iPad.

### ערך עסקי מרכזי

### מצב נוכחי

המערכת בגרסה 2.2, בייצור מלא ב-Railway. מכסה את כל ליבת הפונקציונליות:

- ציוני רספונסיביות לחברות ביטוח

## 1. סקירה כללית

Orly Medical היא מערכת ניהול אירוע רפואי (Medical Case Management) המיועדת למנהלי אירוע רפואי המלווים מטופלים לאורך תהליכים רפואיים מורכבים. המערכת מנהלת את מסע המטופל מרגע הקליטה ועד ההחלמה, ומספקת ניהול מלא של ביטוחים, תביעות, מסמכים, זרימות עבודה מובנות, ואסטרטגיה פיננסית-רפואית.

## 2. מודולים קיימים

### 2.1 אימות ואבטחה

- הרשמה והתחברות עם JWT (תוקף 30 יום)
- אימות דו-שלבי (2FA) — TOTP (Google Authenticator) ואימייל
- שחזור סיסמה עם קישור בתוקף 1 שעה
- ניהול פרופיל משתמש (שם, אימייל, סיסמה, מפתח 2FA)
- הגנת נתיבים — ProtectedRoute עם הפניה אוטומטית לדף כניסה
- כניסה בלחצן יחיד בנאב-בר עם מודל "התחברות / הרשמה"

### 2.2 ניהול מטופלים

- יצירה, עריכה ומחיקה של מטופלים
- שיוך קופת חולים ורמת ביטוח (בסיסי / משלים / פרמיום / זהב)
- פרטי אבחנה, מצב רפואי, ותגיות מצב רפואי (condition tags)
- חיפוש וסינון דינמי
- 5 טאבים לכל מטופל: פרטים, ביטוחים, תביעות, אסטרטגיה, מסמכים

### 2.3 מסע מטופל (Patient Journey)

ציר זמן אחוד המציג את כל שלבי הטיפול הרפואי של המטופל:

- **6 שלבים קבועים:** קליטת מטופל — גילוי ואבחון — תכנון הטיפול — שלב הטיפולים — החלמה ושיקום — מעקב
- **צמתים מותאמים אישית:** ניתן להוסיף אירועים בין השלבים (ניתוח, ייעוץ, תביעה...)
- **עריכה inline:** תאריך, הערות, מיקום מחדש בין השלבים
- **סטטוס לכל צומת:** עתידי / פעיל / הושלם
- **סנכרון עם מנוע הזרימה:** השלמת שלב — עדכון אוטומטי של הציר

### 2.4 ביטוחים (Insurance)

- **מקורות ביטוח:** סל הבריאות, קופת חולים, הר הביטוח, ביטוח פרטי, ביטוח לאומי
- **ייבוא אוטומטי:** סל הבריאות, קופת חולים, ביטוח לאומי, הר הביטוח (Excel), PDF פוליסה פרטית
- **קטגוריות כיסוי:** ניתוח, אשפוז, שיקום, אבחון, השתלה, מחלות קשות, טכנולוגיה מתקדמת, חוות דעת שנייה
- **פרטי כיסוי:** סכום מכוסה, אחוז כיסוי, השתתפות עצמית, תקרה שנתית, כיסוי בחו"ל
- **כיסויים מותאמים אישית:** הוספת קטגוריות ייחודיות מחוץ לרשימה הסטנדרטית
- **יועץ כיסוי אוטומטי:** חישוב כיסוי לכל שלב בזרימת העבודה

### 2.5 תביעות (Claims)

- יצירה ומעקב תביעות מול כל מקורות הביטוח
- **סטטוסים:** טיוטה / ממתין / הוגש / אושר / חלקי / נדחה
- **יצירה אוטומטית** של תביעות טיוטה ע"י מנוע הזרימה לשלבים פיננסיים
- שדות: סכום מבוקש, סכום מאושר, תאריך הגשה, דדליין, עדיפות
- קישור ישיר לשלב בזרימת עבודה
- **Learning Feedback:** עדכון ציוני רספונסיביות מתוצאות תביעות

### 2.6 מסמכים (Documents)

- העלאת מסמכים לפי מטופל (PDF, תמונות, Word, Excel — עד 20MB)
- קטגוריות: רפואי, ביטוחי, משפטי, פיננסי, אחר
- הורדה ומחיקה מאובטחת
- אחסון בנפח מתמיד: /data/uploads/{patient_id}/ על Railway

### 2.7 מאגר רופאים

- כ-370 רשומות ממקורות מגוונים: CKAN ממשלתי, Google Sheets, אתרים רפואיים
- נרמול אוטומטי: שם, מומחיות, תת-מומחיות, טלפון, מיקום
- סינון לפי מומחיות, קבלת קופ"ח, מתן חוות דעת מומחה
- ייבוא PDF / Excel, ייצוא Excel עם עיצוב RTL ועברית
- חיפוש חופשי בשם, עיר ומומחיות

### 2.8 ציוני רספונסיביות

- דירוג חברות ביטוח וקופות חולים לפי איכות שירות
- מדדים: מהירות תגובה (1-10), רמת בירוקרטיה (1-10), ציון כולל
- עדכון אוטומטי מ-Learning Feedback של תביעות מאושרות / נדחות
- משמש לדירוג מקורות ביטוח מומלצים בזרימות העבודה

### 2.9 אסטרטגיה ייעוצית

- ניהול זכאויות: קיימות, פוטנציאליות, צפויות
- המלצות תביעות לפי שלב טיפולי ומצב המטופל
- מטריצת כיסויים — סיכום ויזואלי של כל הביטוחים
- מנוע המלצות זרימות — מציע תהליכים רלוונטיים לפי תגיות המצב הרפואי

## 3. מנוע זרימת עבודה (Flow Engine)

מנוע הזרימה הוא ה-State Machine המנהל תהליכים מובנים לכל מטופל.

### 3.1 מושגי יסוד

| מושג | תיאור |
|------|--------|
| Workflow Template | תבנית זרימה ניתנת לשימוש חוזר |
| Workflow Instance | הרצה ספציפית של תבנית עבור מטופל |
| Step | שלב: ממתין / פעיל / הושלם / דולג |
| Task | משימת checklist — חובה לפני מעבר שלב |
| Audit Log | רשומת כל פעולה — בלתי ניתן למחיקה |
| Coverage Advisor | חישוב כיסוי ביטוחי אוטומטי לכל שלב |
| Force Advance | מעבר מאולץ — עוקף משימות, נרשם בלוג |

### 3.2 לוגיקת State Machine

| פעולה | מצב לפני | מצב אחרי | תנאי |
|--------|-----------|-----------|------|
| advance_step() | active | active הבא | כל המשימות הושלמו |
| advance_step(force=True) | active | active הבא | ללא בדיקת משימות + לוג |
| skip_step() | active | skipped | שלב אופציונלי בלבד |
| pause_instance() | active | paused | תמיד |
| resume_instance() | paused | active | תמיד |
| cancel_instance() | active/paused | cancelled | עם סיבה |

### 3.3 תבניות מובנות

| תבנית | שלבים | נוצר אוטומטית |
|--------|-------|----------------|
| מסע המטופל | 6 | עם יצירת כל מטופל |
| הגשת תביעה ביטוחית | 5 | ידנית |
| ערר על דחיית תביעה | 7 | ידנית |
| קבלת חוות דעת שנייה | 6 | ידנית |
| הגשה לביטוח לאומי | 7 | ידנית |
| תהליך אשפוז מתוכנן | 9 | ידנית |
| פתיחת תיק אונקולוגי | 7 | ידנית |
| מעקב טיפול כימותרפי | 5 | ידנית |
| הכנה להשתלת איבר | 7 | ידנית |
| שיקום לאחר שבץ מוחי | 6 | ידנית |

### 3.4 Coverage Advisor

בכל הפעלת שלב חדש המערכת מחשבת אוטומטית:

- התאמת קטגוריות כיסוי לביטוחים הפעילים של המטופל
- חישוב: סכום מכוסה, פער (gap), אחוז כיסוי לכל מקור
- דירוג מקורות לפי ציוני רספונסיביות
- המלצה בעברית — מה לעשות ומאיזה מקור
- יצירת תביעת טיוטה אוטומטית לשלבים פיננסיים עם כיסוי

## 6. API עיקרי

### Auth

- POST /api/auth/register — הרשמה
- POST /api/auth/login — כניסה + 2FA
- POST /api/auth/verify-2fa — אימות קוד
- POST /api/auth/forgot-password — שחזור סיסמה

### Patients

- GET/POST /api/patients — רשימה / יצירה
- GET/PUT/DELETE /api/patients/{id} — ניהול מטופל
- GET/POST/PUT/DELETE /api/patients/{id}/nodes — צמתים

### Workflows

- GET /api/workflows/templates — תבניות
- POST /api/workflows/instances — הפעלת זרימה
- POST /api/workflows/instances/{id}/steps/{step_id}/advance — השלמת שלב
- POST /api/workflows/instances/{id}/steps/{step_id}/tasks/{task_id}/toggle — סימון משימה
- POST /api/workflows/instances/{id}/pause | resume | cancel

### Insurance / Claims / Documents

- GET/POST /api/patients/{id}/insurance — ביטוחים
- GET/POST /api/patients/{id}/claims — תביעות
- POST /api/patients/{id}/documents — העלאת מסמך
- GET /api/patients/{id}/documents/{doc_id}/download — הורדה
